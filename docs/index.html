이 프로그램은?

복잡한 방송 프로그램(OBS) 없이, PC를 켜두는 것만으로 수업/현장을 자동으로 기록·방송하도록 만든 도구입니다.

🔴 자동 송출

PC를 켜면 약 30초 뒤 알아서 유튜브 라이브(일부 공개)가 시작됩니다. 별도 클릭이 필요 없습니다.

💾 로컬 백업 녹화

같은 화면을 내 PC 폴더에도 저장합니다. 인터넷이 끊겨도 녹화는 계속됩니다.

📱 폰에서 확인·제어

폰 브라우저로 상태 확인, 시작/중지, 교시 시간표 입력을 할 수 있습니다(앱 설치 불필요).

화면 좌측 상단의 작은 점 = 상태 신호등

🟢 정상 송출 중 🟡 연결 중 · 준비 중 🔴 오류 · 재시도 중 (녹화는 계속)

본인 소유 PC의 자기 기록·방송 목적 전용입니다. 타인의 화면·음성을 동의 없이 촬영하는 용도로는 사용할 수 없습니다(이용약관).

시스템 요구사항

운영체제Windows 10 / 11 (64비트) 전용
런타임설치 불필요 — .NET 8 · FFmpeg · cloudflared가 모두 포함(자체 포함 배포)되어 있습니다.
그래픽카드필수 아님. NVIDIA(NVENC) → AMD(AMF) → Intel(QSV) → CPU(x264) 순으로 자동 선택됩니다. GPU가 있으면 CPU 부담이 줄어듭니다.
디스크녹화 저장용 여유 큰 드라이브 권장(기본 한도 100GB, 여유 5GB 미만이면 녹화 자동 일시정지).
계정라이브가 가능한 YouTube 채널(아래 사전 활성화 참고).
인터넷송출/업로드에 필요. 끊겨도 녹화는 계속되고, 복구되면 자동 재연결됩니다.

다운로드 시 보안 경고가 뜨는 것은 정상입니다

개인이 만든 서명 없는 프로그램이라 Windows가 경고를 표시합니다. 아래 설치하기의 안내대로 [추가 정보] → [실행]을 누르면 됩니다.

설치하기

설치는 설정 마법사 없이 한 번에 끝나고, 끝나면 앱이 자동으로 실행됩니다.

  1. 1
    내려받은 Setup.exe를 더블클릭합니다.
  2. 2
    “Windows의 PC 보호” 창이 뜨면 → 추가 정보를 누른 뒤 나타나는 실행 버튼을 클릭합니다. (오른쪽 그림 참고)
  3. 3
    설치가 자동으로 진행되고, 끝나면 앱이 스스로 실행됩니다. 선택할 화면(마법사)은 없습니다.
  4. 4
    자동 시작 방식·녹화 폴더 등 세부 설정은 나중에 제어판(Ctrl+Shift+F12) → 설정에서 바꿉니다.
Windows의 PC 보호
Microsoft Defender SmartScreen에서 인식할 수 없는 앱의 시작을 차단했습니다. 이 앱을 실행하면 PC가 위험에 노출될 수 있습니다.

최초 실행 — 사용 동의

처음 실행하면 이 프로그램이 무엇을 자동으로 하는지 알려주는 동의 화면이 한 번 나타납니다.

“Media Capture Helper 사용 동의” 창은 다음 4가지 자동 동작을 안내합니다.

  • 주 모니터 화면 전체 캡처(마우스 커서 포함)
  • 시스템 소리 + 마이크 캡처·혼합
  • 캡처한 내용을 내 유튜브 계정으로 일부 공개 라이브 송출
  • 같은 내용을 로컬 폴더에 백업 녹화

체크박스를 선택한 뒤 동의하고 시작을 누르면 시작됩니다. 동의하지 않으면 프로그램이 종료됩니다.

그다음은 자동입니다 — 사용자가 할 일 없음

인증 파일 배치, 연결된 마이크 자동 등록, 자동 시작 등록이 알아서 이뤄지고, 약 30초 뒤(네트워크 준비) 자동으로 송출+녹화가 시작됩니다.

YouTube 연결 (최초 1회)

첫 송출 때 브라우저가 한 번 열립니다. 본인 Google 계정으로 로그인하면 끝이며, 이후에는 자동으로 갱신되어 다시 로그인할 필요가 없습니다.

설치 전날, 채널의 “실시간 스트리밍”을 미리 켜두세요

채널에 라이브 스트리밍이 활성화돼 있어야 송출됩니다. YouTube 스튜디오 → 만들기 → 실시간 스트리밍에서 켤 수 있으며, 최초 활성화에 최대 24시간이 걸릴 수 있습니다.

로그인이 계속 막힌다면, 배포용 Google 프로젝트가 “테스트” 상태라 등록된 테스트 사용자만 로그인할 수 있는 경우입니다 — 부록을 참고하세요. 로그인 정보(토큰)는 Windows DPAPI로 암호화되어 이 PC에만 저장됩니다.

매일 사용법

PC를 켜면 끝입니다. 약 30초 후 자동으로 송출과 녹화가 시작됩니다.

상태 확인

화면 좌측 상단의 6px 점(기본 숨김, 설정에서 켤 수 있음)으로 상태를 봅니다.

정상 송출 연결 중 재시도 중

알아두면 좋은 점

  • 라이브는 항상 일부 공개(Unlisted) — 링크를 아는 사람만 볼 수 있습니다.
  • PC를 끄면 방송이 종료되고 녹화 파일이 안전하게 마감됩니다.
  • 송출이 끊겨도 녹화는 계속되고, 네트워크가 복구되면 자동 재연결됩니다.

제어판 둘러보기

송출 품질 (자동 절약)

별도 설정 없이도, PC 부하나 네트워크가 나빠지면 송출 화질을 자동으로 한 단계씩 낮춰 끊김을 막고, 여건이 회복되면 쉬는 시간에 원본으로 되돌립니다. (소리는 절대 낮추지 않습니다.)

자동 모드 (기본)

인코딩이 버거우면(EncodeOverload) 또는 업링크가 부족하면(NetworkCongestion) 원본 → 절약 1단계 → 2단계…로 낮췄다가, 안정되면 자동 복귀합니다.

수동 고정

폰 원격(또는 제어판)에서 원하는 단계로 고정할 수 있습니다. 고정하면 자동 변경이 멈추고, 방송을 전체 중지하면 자동 모드로 복귀합니다.

폰 원격의 “송출 품질” 카드에 현재 단계(예: 절약 1단계 · 자동)가 표시됩니다.

호실명으로 제목 구분

여러 PC가 한 채널에 송출할 때, 각 PC에 호실명을 넣으면 제목 앞에 [201호]가 자동으로 붙어 서로 구분됩니다.

  1. 1
    제어판 → 설정 → 호실명에 방 이름(예: 201호)을 입력하고 설정 저장.
  2. 2
    다음 방송/교시부터 라이브·VOD 제목 앞에 [201호]가 자동으로 붙습니다.

제목 예시 — 라이브: [201호] 라이브 - 2026-07-13 09:00 · 교시 VOD: [201호] 1교시 - 2026-07-13. 호실명을 비우면 [ ] 부분은 생략됩니다.

여러 PC가 한 유튜브 채널에 송출하려면

채널을 브랜드 계정으로 만들고 각 방 운영자 Google 계정을 관리자(매니저)로 추가한 뒤, 각 PC에서 그 계정으로 로그인해 공유 채널을 선택합니다. ⚠️ 업로드 한도는 Cloud 프로젝트 단위로 공유되므로, PC가 많으면 프로젝트 분리 또는 할당량 증설을 권장합니다.

교시 자동 분할 VOD

하루 종일 라이브 하나는 그대로 두면서, 교시 시간표에 맞춰 녹화본을 교시별로 잘라 “N교시 - 날짜” 제목의 일부 공개 영상으로 유튜브에 자동 업로드합니다. 쉬는 시간은 올라가지 않습니다.

동작 방식

  1. 1
    시간표를 입력합니다(폰 원격 또는 config.json). 요일별 기본값 + “오늘만 덮어쓰기”(단축수업 등, 다음 날 자동 해제)를 지원합니다.
  2. 2
    각 교시 종료 시각에 녹화본에서 해당 구간을 무손실(재인코딩 없음)로 잘라냅니다. 시작점은 키프레임 기준 최대 2초 어긋날 수 있습니다(수업 녹화엔 무해).
  3. 3
    잘린 파일은 업로드 큐에 쌓여 순서대로 올라갑니다(재부팅 후에도 유지).

업로드 한도(하루 약 6건)와 다음날 이월

유튜브 업로드 API는 하루 약 6건까지 허용됩니다. 7교시 이상이면 초과분이 큐에 보존됐다가 다음 날 자동 업로드됩니다(리셋 ≈ 한국시간 16~17시). 매일 당일에 모두 올리려면 할당량 증설을 신청하세요: Google Cloud Console → YouTube Data API → Audit and Quota Extension(무료, 심사 수일~수주). 좁은 회선이면 uploadTiming을 after-hours(방과 후 일괄)로 바꿀 수 있습니다.

텔레그램 알림 (선택)

문제가 생기면 폰으로 즉시 알림을 받을 수 있습니다 — 마이크 무음 · 송출 끊김 · 디스크 부족 · 업로드 실패 등.

  1. 1
    텔레그램에서 @BotFather로 봇을 만들고 봇 토큰을 복사합니다.
  2. 2
    알림 받을 채팅 ID를 확인합니다.
  3. 3
    제어판 → 설정 → 텔레그램 알림에 토큰과 채팅 ID를 붙여넣고, 알림 수준(정보/경고/심각)을 고른 뒤 테스트 알림 보내기로 확인 → 설정 저장.

봇 토큰은 개인 정보이니 공유하지 마세요. 저장 시 자동으로 암호화됩니다.

폰 원격 제어

같은 와이파이(또는 외부)의 폰 브라우저로 접속해 라이브 시작/중지, 상태·미리보기·오디오·송출 품질 확인, 교시 시간표 편집, 업로드 큐 확인을 할 수 있습니다. 폰에 앱을 설치할 필요가 없습니다.

먼저 원격을 켜야 합니다 (기본은 꺼짐)

config.json의 remote.mode를 "lan"(또는 tailscale/cloudflare)으로 바꾸고 앱을 재시작합니다(포트 8787).

파일 위치: 탐색기 주소창에 %AppData%\MediaCaptureHelper 를 입력해서 엽니다.

연결(페어링)

  1. 1
    폰 브라우저에서 http://<PC의 LAN IP>:8787 접속.
  2. 2
    제어판(Ctrl+Shift+F12)에 표시된 6자리 PIN을 폰에 입력 → 연결.
  3. 3
    한 번 연결하면 기기 토큰이 저장되어 다음부터는 PIN 없이 접속됩니다.

폰 원격 — 외부 접속 (고급)

집·외부에서, 같은 와이파이가 아니어도 접속하는 방법입니다. 상황에 맞는 방식을 고르세요.

방식특징추천 상황
LAN같은 와이파이, 평문(HTTP)같은 공간에서 간단히
Tailscale암호화, 설치 필요(PC+폰)집 밖에서 안전하게
Cloudflare 명명형 터널고정 HTTPS 주소, 방화벽·포트포워딩 불필요학교 PC에 매일 같은 주소로(도메인 연 ~1.5만원)
Cloudflare 고정 주소(명명형 터널) 8단계 설정
  1. Cloudflare 무료 계정 가입(이메일 인증).
  2. 도메인 확보: Domain Registration → Register Domains에서 새로 구매(가장 간단, 자동 연결) 또는 기존 도메인을 Add a site로 연결.
  3. (기존 도메인이면) 상태가 Active가 될 때까지 대기.
  4. Zero Trust → Networks → Tunnels → Create a tunnel → 커넥터 Cloudflared 선택 → 터널 이름 저장 → 화면의 긴 토큰 문자열만 복사(설치는 앱이 알아서 함).
  5. Public Hostname → Add: Subdomain 지정 · Type HTTP · URL localhost:8787 저장 → 폰 주소가 https://이름.내도메인.com으로 고정.
  6. 현장 PC의 config.json(%AppData%\MediaCaptureHelper) remote 항목을 아래처럼 맞춤.
  7. 앱 재시작 → 붙여넣은 토큰이 자동으로 암호화되고, 제어창에 고정 주소 + 6자리 PIN이 표시됨.
  8. 폰 브라우저(셀룰러/와이파이 무관)로 그 고정 주소 접속 → PIN 입력 → 완료(다음부터 PIN 불필요).
"remote": {
  "mode": "cloudflare",
  "port": 8787,
  "cloudflareTunnelToken": "여기에_4단계에서_복사한_긴_토큰_붙여넣기",
  "cloudflareHostname": "이름.내도메인.com"
}

deviceTokens 같은 다른 줄이 있으면 건드리지 말고 그대로 둡니다. 저장 인코딩은 UTF-8.

녹화 파일 관리

위치기본 %USERPROFILE%\Videos\MediaCaptureHelper (제어판에서 변경 가능)
파일명부팅~종료가 1개 파일. 예 REC_2026-07-13_0930.mp4 (인코더 재시작 시 _part2 추가)
자동 정리보존 7일 초과분과 100GB 초과분을 오래된 것부터 삭제(둘 다 조정 가능)
안전장치남은 디스크 5GB 미만이면 녹화 일시정지. 갑작스런 전원 차단에도 파일은 재생 가능

자동 업데이트 · 제거

자동 업데이트

6시간마다 조용히 자동 업데이트를 확인하고, 다음 재시작 때 적용합니다. 제어판 하단 상태바에 “재시작 시 적용” 안내가 뜹니다.

제거

Windows 설정 → 앱에서 Media Capture Helper 제거. 자동 시작·방화벽 규칙은 함께 해제되고, 녹화 파일과 설정은 삭제되지 않습니다.

문제 해결 / 자주 묻는 질문

설치할 때 “Windows의 PC 보호” 경고가 떠요
정상입니다. 서명 없는 개인 배포라서 표시됩니다. [추가 정보] → [실행]을 누르세요.
상태 점이 계속 🟡(연결 중)이에요
인터넷 연결과 YouTube 라이브 스트리밍 활성화 여부를 확인하세요(최초 활성화에 최대 24시간).
상태 점이 계속 🔴(재시도)이에요
%AppData%\MediaCaptureHelper\logs\의 최신 로그를 확인하세요. 프로그램은 자동으로 무한 재시도하며, 그동안 녹화는 계속됩니다.
로그인 브라우저 창이 안 떠요
설정 폴더에 client_secret.json 파일이 있는지 확인하세요(파일 존재 여부만 확인, 내용은 열 필요 없음).
마이크 소리가 안 나가요
제어판 → 오디오 믹서에서 장치 선택·증폭·게이트를 확인하세요. 마이크가 없어도 시스템 소리는 송출됩니다.
녹화는 되는데 송출이 안 돼요
네트워크 문제일 때의 정상 동작입니다. 복구되면 자동으로 재연결됩니다.
게임 전체화면에서 검은 화면이에요
자동 재초기화를 기다리거나, 창 모드(테두리 없는 전체화면)를 사용하세요.
단축키가 안 먹혀요
다른 프로그램과 충돌한 경우입니다. 제어판 설정에서 다른 조합으로 바꾸세요.
교시 영상이 그날 다 안 올라가요
업로드 한도(하루 약 6건) 때문입니다. 초과분은 다음 날 자동 업로드됩니다(정상). 매일 당일 업로드하려면 할당량 증설을 신청하세요.

이용약관 (요약)

✅ 허용

본인이 소유·관리하는 PC에서 본인의 화면·음성을 기록·방송하는 목적.

🚫 금지

타인 PC에 무단 설치, 타인의 화면·음성·개인정보를 동의 없이 캡처·송출, 감시·스토킹 등.

🔒 데이터

녹화는 로컬에만 저장(개발자에게 전송 안 됨). 유튜브는 본인 계정으로만. 토큰은 DPAPI 암호화.

전체 약관 보기 (EULA)

1. 소프트웨어의 동작 — 주 모니터 전체 캡처(커서 포함), 시스템·마이크 오디오 혼합, 본인 유튜브 계정 일부 공개 송출, 로컬 백업 녹화(기본 7일), 부팅 시 자동 시작.

2. 허용되는 사용 — 본인 소유·관리 PC에서 본인의 화면·음성을 기록·방송하는 목적에 한정.

3. 금지되는 사용 — 타인 PC 무단 설치, 타인의 화면·음성·개인정보 무단 캡처·송출·저장, 감시·스토킹·정보 탈취 등 관련 법령(개인정보 보호법, 통신비밀보호법 등) 위반 행위. 위반 책임은 전적으로 사용자에게 있음.

4. 데이터 처리 — 녹화 파일은 지정한 로컬 폴더에만 저장되고 개발자에게 전송되지 않음. 유튜브 송출은 사용자가 직접 로그인한 본인 계정으로만 이뤄짐. OAuth 토큰은 Windows DPAPI로 암호화되어 사용자 프로필에 저장됨.

5. 보증의 부인 — 소프트웨어는 “있는 그대로” 제공되며, 데이터 손실·방송 사고·법적 분쟁 등 결과에 대해 개발자는 책임지지 않음.

6. 동의 철회 — 동의하지 않으면 제거하면 됨. 제거 시 자동 시작 등록이 해제됨.
